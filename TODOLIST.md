# 项目 TODOLIST（PC 优先版）

目标：
- 先把 PC 端做成“前期紧张、中期成型、后期爆发”的完整爽游闭环。
- 采用低成本协作：`GLM-4.7 / GLM-5` 先做基础实现，我负责审查、重构、平衡和复杂机制。

原则：
- 每个模块都走 `实现 -> 自测 -> 审查 -> 合并`。
- 优先可扩展，不做一次性硬编码。

---

## A. 架构与工程（P0）

- [ ] 目录结构固定：`src/entities`、`src/systems`、`src/data`、`src/config`、`src/scenes`、`src/ui`
- [ ] 统一配置层：玩家、武器、怪物、Boss、掉落、升级池全部数据化
- [ ] 建立状态机：`GameState`（开局/进行中/升级中/暂停/结算）
- [ ] 事件命名规范化：`game:*`、`combat:*`、`ui:*`
- [ ] 增加 `CHANGELOG.md` 与版本号

验收：
- 新增武器或怪物时，不改主循环，只改配置+注册。

---

## B. 核心循环（P0）

- [ ] 开局 0-2 分钟：高压但可活
- [ ] 中期 2-6 分钟：构筑成型，开始爽感
- [ ] 后期 6+ 分钟：怪海+精英+Boss 叠压，玩家靠 Build 破局
- [ ] 升级/被动选择时全局暂停（怪、弹幕、计时全部冻结）
- [ ] 受击与击杀反馈强化（震屏、飘字、音效、特效）

验收：
- 玩家不会出现“选升级时被打死”的体验问题。

## I. GLM 低成本协作分工（重点）

### I1. 用 GLM 做什么（低风险、重复性高）
- [ ] 新增配置项（武器表/怪物表/文案）
- [ ] 常规 UI 布局微调
- [ ] 简单功能脚手架（新被动占位、新升级项占位）
- [ ] 非关键模块重命名、拆文件、加注释

### I2. 不交给 GLM 的任务（高风险）
- [ ] 核心战斗循环与碰撞逻辑
- [ ] 进化判定与联动链路
- [ ] 概率与数值平衡
- [ ] 性能热点与抖动修复
- [ ] Boss 机制与阶段状态机

### I3. 建议工作流（省钱且稳）
1. GLM 先产出初稿（只做单模块，不跨太多文件）
2. 本地自测（至少 build 通过）
3. 我做代码审查（逻辑漏洞、边界、性能、可维护性）
4. 我做第二轮优化（重构、联动、数值和手感）
5. 回归测试后再合并

### I4. 给 GLM 的任务模板（可直接复制）

```txt
你只修改以下文件：<文件路径列表>
目标：<单一目标>
限制：
1) 不改核心主循环
2) 不改事件名和数据结构
3) 不新增全局状态
4) 修改后必须保证 npm run build 可通过
输出：
1) 修改了哪些文件
2) 每个文件改了什么
3) 还存在哪些风险点
```

### I5. 审查闸门（我来卡）
- [ ] 是否破坏已有玩法
- [ ] 是否引入隐藏耦合
- [ ] 是否可数据驱动扩展
- [ ] 是否有性能回退
- [ ] 是否符合 PC 端读感与爽感目标
